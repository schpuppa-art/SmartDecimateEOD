# SmartDecimateEOD v6.0

AviSynth+ x64 плагин. Content-aware дециматор для анимации: выбрасывает
дублирующиеся кадры так, чтобы общая длина потока стала ровно `fps_out` (CFR,
безопасно для звука), но настоящая нарисованная анимация не пострадала. Вход —
**только YV12**.

Новое в v6.0: дециматор читает `scuts.avs.scenes.txt` **сам** через `scenes_file=`;
отдельный `SCDetectEOD`-клип больше не нужен. Fade-детекция в file-режиме работает —
meanY считается покадрово из thumbnail. Старые скрипты с `sc`-клипом — байт-в-байт.

## Зачем нужен

Анимация неравномерна по «частоте рисунка». Есть сцены на **двойках**
(`O D D D D` — оригинал, дальше дубли) и на **единицах** (`O O O O O` — каждый
кадр новый, движение нарисовано полностью). Классический дециматор режет по
фиксированному коэффициенту на чанк («выбрасывай каждый N-й») и на анимации
уничтожает реальное движение, потому что не отличает дубль от оригинала.

SmartDecimateEOD держит **глобальный бюджет**: total output count фиксирован
через `fps_out`, но распределение оставленных кадров по сценам адаптивно к
содержимому. Дубли из сцен на двойках работают как «кредит» — их выброс
освобождает бюджет, который тратится на сохранение **всех** оригиналов в сценах
на единицах. Длина итога фиксирована, режется только лишнее.

## Архитектура: two-pass eager preanalysis

Весь анализ выполняется **один раз в конструкторе** (stateless, переживает
пересборку графа в AvsPmod). GetFrame после этого работает по индексной таблице
без обращения к декодеру.

**Pass 1 — классификация (тяжёлая).** Декодит каждый кадр, отвечает на вопрос
«оригинал или дубль» для каждого. Делится на две фазы:
- **Pass 1A** — один проход декода: SSIM-thumbnail (160×120, box-average Y,
  studio→full, 3×3 blur) против предыдущего кадра. Заодно (в file-режиме)
  считается meanY кадра из thumbnail.
- **Pass 1B** — по сценам: SSIM-guard + регистрация (SAD ±3) + block-ratio.
  Регистрация **отложенная** — гоняется только на сценах, прошедших дешёвый
  guard; static / on-ones / fast-motion / fade сцены её пропускают. Это и держит
  Pass 1 быстрым на смешанном материале (×2–4 ускорение).

Результат Pass 1 — массивы `is_dup[]`, `resid[]`, `scene_of[]`, `d_alternation[]`.

**Pass 2 — распределение бюджета (лёгкая).** Кадры **не трогает** (ноль
GetFrame). Берёт готовые массивы Pass 1 и решает, какие кадры оставить, чтобы
попасть ровно в `fps_out`: сохраняет все ORIG, остаток бюджета заполняет первым
DUP после каждого ORIG (сохраняет локальный ритм двоек), error-diffusion между
сценами для точного попадания в total.

Pass 2 — **потребитель** массивов Pass 1, не повтор работы. Слить их в один
проход по кадрам нельзя: бюджет глобален (нужна сумма ORIG/DUP по всем сценам),
а классификация локальна (кадр против соседа). Пока Pass 1 не прошёл весь клип,
Pass 2 не знает суммы.

## Источник границ (`scenes_file` приоритетен)

Границы сцен — обязательный вход (на них держится `cfr_per_scene` бюджет, fade
detection). Два источника, по жёсткой иерархии:

1. **`scenes_file`** задан и не пуст — прямое чтение `scuts.avs.scenes.txt`
   (формат SCDetectEOD: число + опциональный `# комментарий`). Парсер: `strtol`
   на начале строки, отброс комментария; кадр 0 и вне `[1, N_in)` игнорируются;
   только строго возрастающие. Ноль frame-prop traffic, `sc`-клип не нужен.
   Чтение через `fopen_utf8` (`MultiByteToWideChar` + `_wfopen`) — кириллические
   пути работают.
2. **`sc`-клип** задан, `scenes_file` пуст — границы через frame-prop
   `_scd_boundary` на каждом кадре (legacy). meanY берётся из `_scd_meanY`.

При наличии обоих `scenes_file` выигрывает, `sc` игнорируется. Если не задан ни
один — ошибка (границы нужны жёстко).

## Fade-детекция в обоих режимах

Fade (плавное затемнение/осветление) — последовательность, где яркость
**монотонно** ползёт. Детектор считает дрейф `|meanY[конец сцены] − meanY[начало]|`
и долю нарушений монотонности **внутри** сцены. Прошло пороги (`fade_mean_drift`,
`fade_ratio`) → сцена форсится в all-ORIG (`protect_fade`), фейд не прорежается.

Детектору нужен meanY **на каждом кадре** сцены, не только на границах.

- **sc-режим:** `_scd_meanY` приходит покадрово с клипа (SCDetectEOD как фильтр
  ставит проп на всех кадрах).
- **file-режим (v6.0):** в `scenes.txt` покадрового meanY нет (это разрежённый
  список границ). Поэтому meanY считается из thumbnail в Pass 1A — кадр и так
  декодится для SSIM, `mean(thumb)` бесплатен. Шкала (full-range Y) совпадает с
  `_scd_meanY`, пороги fade переносятся без изменений.

Без этого (в v5.x file-режиме его не было) fade-детектор был бы отключён и на
фейдах шёл бы рывок.

## Кэш и режимы

Анализ кэшируется в `cache=` (бинарный файл с хедером и всеми диагностическими
массивами). Two-pass workflow: prep-скрипт прогоняет анализ и наполняет кэш,
production-скрипт грузит мгновенно.

Режимы (`mode`) и их `mode_id` для ключа кэша:
- **`cfr` / `cfr_per_scene`** → `mode_id=0`. Per-scene CFR: каждая сцена держит
  `round(len × fps_out / fps_in)` кадров. Обе строки эквивалентны для кэша.
- **`content`** → `mode_id=1`. Выбрасывает все дубли без бюджета.
- **`mark_only`** → `mode_id=2`. Кадры не режет (`N_out=N_in`), только пишет
  диагностические props (`_sdec_bypass` и др.). Для гибридного RIFE-пайплайна.

**Ключ кэша = хеш параметров.** Хеш считается **по полям поимённо**
(`params_hash_for`), не по сырой памяти структуры — иначе padding-байты между
`double`/`int`/`uint8_t` дают недетерминированный промах при идентичных
параметрах (исправлено в v5.9).

**`mark_only` грузит кэш, записанный `cfr`/`cfr_per_scene`** (ищет под
`mode_id=0`, не под своим `mode_id=2`). Поэтому хеш-параметры обоих вызовов
обязаны совпадать:

> **`fps_out` входит в хеш.** Если prep-вызов задал `fps_out=12`, а `mark_only`
> его не задал — берётся дефолт `12.5`, хеш не совпадает, `mark_only` падает в
> fallback и гонит полный Pass 1 заново. Это самый частый источник «второго
> полного прогона». `fps_out` в `mark_only` указывать **обязательно**, хотя длину
> он там не меняет.

В хеш **не входят**: `cache`, `log`, `scuts_out`, `scenes_file`, `analyze_log` —
их дублировать между вызовами не нужно.

## Гибридный RIFE-bypass

`mark_only` пишет проп `_sdec_bypass` на сцену: `alternation ≥ bypass_alt` И
`sceneLen ≥ bypass_min_len` → сцена на двойках, годится на RIFE-интерполяцию
(`bypass=0`); иначе берётся source (`bypass=1`). Fade-сцены принудительно
`bypass=1` (мало кадров для надёжной интерполяции). Bypass только **по-сценный** —
sub-scene переключение даёт видимые швы на стыках (тестировалось, откатано).

## Параметры

| Параметр | Тип | Дефолт | Назначение |
|---|---|---|---|
| `clip` | clip | — | вход YV12, source (позиционный) |
| `sc` | clip | — | источник границ через `_scd_boundary` (опционален; нужен если нет `scenes_file`) |
| `scenes_file` | string | "" | путь к списку границ; **приоритетен** над `sc` |
| `fps_out` | float | 12.5 | целевая частота; **входит в хеш кэша** |
| `mode` | string | "cfr" | "cfr"/"cfr_per_scene"/"content"/"mark_only" |
| `reg_scale` | int | 1 | downscale регистрации (1=full, 2=half, 4=quarter); в хеше |
| `bypass_alt` | float | 0.80 | порог alternation для RIFE-eligibility; в хеше |
| `bypass_min_len` | int | 4 | мин. длина сцены для RIFE; в хеше |
| `bypass_window` | int | 12 | полу-окно локального alternation |
| `th_ssim_dup` | int | 990 | SSIM-guard: q80 < этого → сцена без дублей (skip регистрации); в хеше |
| `residual_ceiling` | float | 5.0 | потолок abs residual для percentile-guard; в хеше |
| `residual_mult` | float | 2.5 | порог = q_low × mult; в хеше |
| `th_ratio` | float | 10.0 | порог block max/mean ratio; в хеше |
| `full_range` | bool | false | источник full-range Y; в хеше |
| `thumb_w` | int | 160 | ширина thumbnail для SSIM/meanY |
| `thumb_h` | int | 120 | высота thumbnail |
| `fade_ratio` | int | 60 | percent 0..100, допустимая доля немонотонности фейда |
| `fade_mean_drift` | int | 20 | мин. `|meanY[конец]−meanY[начало]|` для фейда |
| `protect_fade` | bool | true | форсить all-ORIG на фейдах |
| `allow_original_cut` | bool | false | разрешить резать оригиналы (если `fps_out` слишком низкий) |
| `cache` | string | "" | путь к бинарному кэшу анализа |
| `log` | string | "" | путь к per-scene текстовому логу |
| `scuts_out` | string | "" | путь к границам на децимированном таймлайне |
| `analyze_only` | bool | false | всегда перепрогон Pass 1 (без кэша), диагностика |
| `analyze_log` | string | "" | путь к analyze-логу |
| `debug` | bool | false | per-frame диагностика в stderr |

## Frame properties

В `mark_only`-режиме (и при `debug`) на каждом кадре:

| Prop | Значение |
|---|---|
| `_sdec_bypass` | 0=RIFE-eligible (двойки), 1=source (всё прочее + фейды) |
| `_sdec_sceneId` | индекс сцены |
| `_sdec_isDup` | 1 если кадр классифицирован как дубль |
| `_sdec_isFade` | 1 если кадр в fade-сцене |
| `_sdec_srcFrame` | исходный номер кадра |

На децимированном выходе (`cfr`-режимы) выставляется `_scd_boundary` на
границах сцен нового таймлайна — для последующих scene-aware фильтров (RIFE с
`sc=true`, SCSignalBorder).

## Примеры

v6.0, прямое чтение файла (без `SCDetectEOD`):
```
src = LWLibavVideoSource("O:\vto2\mikulishna\film.dgi-2.avi")
dec = SmartDecimateEOD(src, fps_out=12, mode="cfr_per_scene", bypass_alt=0.7, \
        reg_scale=2, scenes_file="O:\vto2\mikulishna\scuts.avs.scenes.txt", \
        cache="O:\vto2\mikulishna\sdec.bin", log="...\sdec.log", \
        scuts_out="...\scuts_dec.txt")
```

mark_only для гибридного RIFE (хеш-параметры совпадают с dec, включая `fps_out`):
```
mask = SmartDecimateEOD(src, mode="mark_only", fps_out=12, bypass_alt=0.7, \
        reg_scale=2, scenes_file="O:\vto2\mikulishna\scuts.avs.scenes.txt", \
        cache="O:\vto2\mikulishna\sdec.bin")
```

Legacy через sc-клип (работает как раньше):
```
sc  = src.SCDetectEOD(scenes_file="...\scuts.avs.scenes.txt")
dec = SmartDecimateEOD(src, sc, fps_out=12, mode="cfr_per_scene", cache="...\sdec.bin")
```

## Сборка

```
cl /LD /EHsc /O2 SmartDecimateEOD.cpp
```
Из x64 Native Tools Command Prompt for VS 2022. Старый DLL из `plugins64`
удалить перед заменой.

## Границы применимости

- `SCDetectEOD scenes_file` **не переиспользовать** на децимированном клипе —
  номера кадров сдвигаются. Для детекции на децимированном выходе: `scuts_out`
  даёт уже пересчитанные границы.
- Prep- и production-скрипты обязаны использовать идентичные хеш-параметры,
  иначе кэш-промах.
- `cfr` / `cfr_per_scene` эквивалентны для кэша (один `mode_id`).
- При слишком низком `fps_out` (бюджет меньше числа оригиналов) — ошибка с
  рекомендацией поднять `fps_out` или задать `allow_original_cut=true`.

## Совместимость

Дефолт `mode="cfr"`, `sc` опционален, `scenes_file` именованный в конце
сигнатуры. Скрипты под v5.x с двумя клипами работают байт-в-байт: второй клип
читается как `sc`, `scenes_file` пуст → legacy-путь.
